# Tool Routing

- Use Figma context first when a live design exists.
- Use `design-references/` for screenshots, state captures, and fidelity notes before coding visible UI.
- Use local repository patterns before external inspiration.
- Use component-variation tooling only when structure is already agreed.
- Use Storybook for component states and Playwright for user flows.
- Use Playwright screenshot assertions for whole-page regression on key surfaces.
- Use a performance budget lane for high-visibility pages or newly added rendering cost.
- Use browser inspection tools when visuals look correct but runtime behavior is suspect.
